# ERP ODM Department Workflow

日期：2026-05-23  
用途：作為前端第一版與後續 API 整合的部門流程基準

## 接單前

```txt
開發需求（研發）
-> 配方選材（研發）
-> 打樣（研發）
-> 送樣（業務）
-> 客戶確認樣品（業務）
-> 供應商報價/合約（採購）
-> 成本試算、營養標示（研發）
-> 客戶報價/合約（業務）
```

## 接單後

```txt
訂單（業務）
-> 生產材料請購/排程（生管）
-> 材料採購（採購）
-> 到貨入庫（倉庫）
-> 材料檢驗（品保）
-> 生產（製造）
```

| 配方選材 | 研發 | R&D / Costing |
| 打樣 | 研發 | R&D / Costing |

## 第一版整合原則

1. 接單前流程先以狀態追蹤、成本試算與報價依據為主，不急著做完整簽核。
2. 接單後流程先以唯讀狀態與阻擋關係為主，再逐步開放請購、工單、入庫、檢驗與報工操作。
3. 供應商報價/合約同時影響 R&D 成本試算與業務客戶報價。
4. 材料檢驗未通過前，倉庫庫存不得成為可用生產庫存。
5. 生管的材料請購/排程是正式訂單轉入生產履約的關鍵中樞。

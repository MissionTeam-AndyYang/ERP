
## 0. 前置作業（必須先執行）

請先在本地 repository 執行以下操作：

1. 切換至 main branch
   git checkout main

2. 更新最新檔案
   git pull origin main

3. 確認以下檔案為最新版本：
   - docs/database/食品管理系統Database Schema_0.0.26.docx
   - docs/database/EWDB_20260526.sql

---

## 若 pull 失敗

需停止文件生成並回報錯誤，不得使用舊版本繼續分析。

## 1. 資料來源
請依據以下兩份來源文件進行資料庫整合：

- docs/database/食品管理系統Database Schema_0.0.26.docx
- docs/database/EWDB_20260526.sql

並產出一份統一資料庫說明文件。

---

## 2. 文件輸出位置

docs/spec/database/index.md

---

## 3. 文件格式要求

使用 Markdown（.md），並需符合以下結構：

### (1) 單一入口文件
- 檔案名稱：index.md
- 作為所有資料表的入口索引

---

### (2) 目錄導覽（Table of Contents）

需於文件最上方提供：

- 資料表清單（A~Z排序）
- 每個資料表需提供內部跳轉連結

範例：

- [Employee](#employee)
- [Member](#member)

---

### (3) 章節結構

每一個資料表需獨立章節，並使用：

## TableName（英文資料表名稱）

且需支援可折疊內容（使用 Markdown `<details>`）。

---

### (4) 資料表排序規則

所有資料表章節需依「英文資料表名稱」進行 A → Z 排序。

---

## 4. 每個資料表欄位格式規範

每個資料表需提供以下欄位說明表：

| 欄位名稱 | 資料型態 | 允許Null | 索引 | 外鍵 | 欄位說明 | 值定義 |
|----------|----------|------|-----|------|----------|----------------|

---

## 5. 欄位規則說明

### (1) 索引欄位(Key)

需標示：

- PK
- UK
- （可同時存在）

---

### (2) 外鍵欄位Foreign Key）

需明確指出：

- 來源欄位
- 參照資料表與欄位

範例：

- employee.department_id → department.id

---

### (3) 值定義(Enum)

若欄位為 enum，需展開列出： 
男 (1)、女 (2)

若非 enum 則填寫：
空白

## 6. 來源文件處理規則

- EWDB_20260526.sql 為「最終 schema 基準」
- Word 文件僅作為補充說明參考
- 若兩者衝突，以 SQL 為準

---

## 7. 輸出內容要求

需完整包含：

- 所有資料表
- 所有欄位
- 所有 FK 關聯
- 所有 enum 定義
- 欄位型別需與 SQL 一致

---

## 8. 額外要求（重要）

- 不可遺漏任何 table
- 不可自行刪減欄位
- 不可改寫欄位名稱
- 僅允許做「整理與文件化」

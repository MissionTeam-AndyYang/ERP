# ERP 品保中心第一版工作區規格

日期：2026-05-23  
基準：EWDB_20260522.sql、EWDB_20260522_WORKFLOW.md、前端核心頁面第一版規劃

## 目標

品保中心第一版先處理「是否可放行」與「品質是否阻擋後續流程」。畫面要讓管理者快速知道今天哪些檢驗批次完成、哪些待判定、哪些批號暫緩入庫/出貨/生產，以及文件是否完整。

依使用者補充的部門流程，接單後「材料檢驗」由品保負責，位於「材料採購 -> 到貨入庫 -> 材料檢驗 -> 生產」之間。未通過材料檢驗的批號不得成為可用生產庫存。

## 第一版範圍

1. 今日檢驗批次總覽
   - 原料收貨檢驗
   - 材料檢驗
   - 首件檢驗
   - 製程抽驗
   - 成品檢驗
   - 出貨前檢驗

2. 放行與阻擋判定
   - 可入庫 / 暫緩入庫
   - 可出貨 / 暫緩出貨
   - 可生產 / 暫停生產
   - 待判定、隔離、返工、報廢狀態

3. 異常與 NCR
   - 不良率
   - 異常原因
   - 責任單位
   - 處理期限
   - 是否影響工單、訂單、庫存或出貨

4. 文件完整性
   - COA
   - 運輸溫度紀錄
   - 首件/成品/原料檢驗紀錄
   - 微生物快篩
   - 收貨與出貨相關文件

## 畫面結構

第一版採用與倉庫、生產、訂單、採購、追溯一致的「狀態總覽 + 工作清單 + 選取明細 + 流程節點」模式。

### 區塊一：KPI 摘要

顯示今日品檢批次、待判定批次、阻擋入庫/出貨批次、文件待補批次。

### 區塊二：工作分頁

分頁包含：

- 檢驗批次
- 放行與阻擋
- 異常/NCR
- 文件完整性

### 區塊三：檢驗清單

欄位包含：

- 檢驗單號 / 批號
- 品項與料號
- 來源單據
- 檢驗類型
- 抽樣數 / 不良數
- 阻擋狀態
- 待判項目 / 文件數
- 判定
- 目前狀態

### 區塊四：選取明細

顯示選定檢驗批次的判定說明、抽樣與不良率、負責人、期限、文件狀態與品質流程節點。

## 與其他核心工作區的連動

- 倉庫：品質未放行時，庫存不得成為可用庫存；隔離品應進入暫緩或隔離視圖。
- 生產：首件或原料檢驗未過時，應阻擋或警示工單生產。
- 訂單：成品或出貨前檢驗未完成時，應進入履約風險。
- 採購：原料文件缺失或品質異常應回饋供應商與採購風險。
- 倉庫：到貨入庫後，材料檢驗結果決定可用庫存、隔離庫存或退回處理。
- 追溯：所有品質判定、文件、批號與工單需成為追溯鏈的一部分。

## 後續需要與使用者確認

1. 不同產品類別的品質放行規則是否一致。
2. 哪些檢驗項目屬於「必須完成才可入庫/出貨」。
3. NCR 是否需要第一版就做到正式開單與簽核，或先以狀態追蹤呈現。
4. 是否需要區分法規/客戶要求文件與內部管理文件。

## 第一版不納入

- 完整實驗室檢驗數據輸入表
- 正式電子簽核流程
- 客訴處理完整模組
- 供應商品質評等演算法
- 自動判定規則引擎

# 食品製造管理系統與硬體設備架構說明

## 一、概述

透過 Android 行動裝置、掃描槍、智能秤及標籤機等設備，結合 Server 端(稱為食品製造管理系統)提供之 REST API，完成食品製造過程中的原物料追蹤、秤重管理、批號管理及生產履歷追溯。

---

# 二、架構

採用 Client / Server 架構。

## Client 端

Client 為安裝於 Android 行動裝置上的 APP。

由現場作業人員操作，負責：

* 讀取標籤條碼
* 讀取智能秤資料
* 執行出入庫作業
* 執行入產出產作業
* 列印標籤
* 暫存離線資料

## Server 端

Server 提供 REST API 服務。

負責：

* 訂單管理
* 任務派送
* 批號管理
* 庫存管理
* 製程管理
* 資料驗證
* 資料儲存

Client 與 Server 透過 REST API 交換資料。

---

# 三、硬體整合

## 掃描槍

APP 可整合標籤條碼掃描設備。

透過掃描標籤條碼取得：

* 客戶/產商
* 料品品項類別(原料.物料.膠捲.在製品.製成品)
* 料品品項名稱
* 重量或數量
* 批號效期
* 批號
* 序號

作為作業驗證及追溯依據。

---

## 智能秤

APP 可整合智能秤設備。

應用於：

### 採購進貨

由Server端決定該料品品項是"秤重"還是"計數"，若為計數，由使用者手動輸入。

### 產製作業

* 餘料品秤重
* 廢料品秤重
* 產製品秤重
* 退料品秤重

---

## 標籤機

APP 可整合標籤列印設備。

於進貨或產製產出時列印標籤。

標籤條碼內容包含：

* 客戶/產商
* 料品品項類別(原料.物料.膠捲.在製品.製成品)
* 料品品項名稱
* 重量或數量
* 批號效期
* 批號
* 序號


---

# 四、料品管理

料品品項包含：
* 原料
* 物料
* 膠捲
* 在製品
* 製成品
* 貨品

> 產製品

---

# 五、料品型態

## 新料品

尚未投入任何製程之料品品項。

例如：

* 原料
* 物料
* 膠捲
* 在製品
* 製成品
* 貨品
---

## 餘料品

製程中經調拌、混合、灌餡等作業形成之中間結合物。

例如：

* 餡料
* 半成品混合物
餘料品應採用新批號資訊，並以新批號進行入庫作業。
---

## 廢料品

製程中因損壞、污染或異常而無法繼續使用之料品。
廢料品應採用新批號資訊，並以新批號進行入庫作業。此類料品需納入耗損管理與追溯範圍。

---

# 六、批號與序號管理

## 批號

批號用於識別進貨或產製所形成之料品品項群組，批號由server端控管產生 。

---

## 序號

同一批號下可存在多個實體包裝單位。

例如：

* 批號：A001
* 共 10 箱

則Client APP建立序號編碼：

例如：
* 00001
* 00002
* ...
* 00010

---

# 七、食品製造流程

食品製造主要包含以下主要製程：

1. 前備
2. 加工
3. 包裝

製造流程如下：

## 步驟一：倉庫發料

依據任務清單中指定批號之原物料或在製品。

透過掃描槍掃描標籤條碼：
* 驗證料品品項
* 驗證批號

此時料品狀態為：

* 新料品

建立出庫紀錄。
---

## 步驟二：入產

將原物料由倉庫移轉至產間。

透過掃描槍：

* 驗證料品
* 驗證批號

建立入產紀錄。
---

## 步驟三：產製產出

製程完成後產生：

* 餘料品 
* 廢料品
* 產製品

---

## 步驟四：標籤產生

每個箱體均建立獨立序號。

依任務給予批號資訊與序號產生標籤。

---

## 步驟五：出產入庫

完成貼標後：

* 產間出產
* 倉庫入庫

建立產製數據紀錄。
---

## 步驟六：退料

已領料但未投入製程之剩餘原物料：

* 保留原批號
* 以原批號重新入庫作業

---

# 九、倉庫作業流程

由倉管人員操作。

APP適用角色：

* 倉庫

## 任務清單取得
APP 透過 REST API 取得：

### 入庫資料
* 相關聯單號(如進貨單/派工單...)
* 料品品項類別
* 料品品項型態(如新料品/餘料品/廢料品)
* 料品品項名稱
* 批號


### 出庫資料
* 相關聯單號(如進貨單/派工單...)
* 料品品項類別
* 料品品項型態(如新料品/餘料品/產製品)
* 料品品項名稱
* 批號
* 序號
* 數量或重量

### 入庫

包含：

* 採購進貨
* 倉庫收料
* 收取餘料品
* 收取廢料品
* 收取產製品
* 訂購退貨


### 出庫

包含：

* 採購退貨
* 倉庫發料
* 訂購銷貨
* 報廢
* 研發打樣
* 公關贈品

---

## 入庫作業流程


## 1.入庫秤重作業

### 秤重確認

秤重取得料品品項重量。

### 列印標籤

產生批號資訊與序號條碼標籤。

### 完成入庫

建立入庫紀錄。

## 2.入庫不秤重作業

### 手動輸入
使用者手動輸入數量。

### 列印標籤
產生批號資訊與序號條碼標籤。

### 完成入庫
建立入庫紀錄。

## 3.入庫不秤重掃碼作業

### 掃描標籤
與今日任務清單進行比對驗證：

* 料品品項名稱
* 批號

### 完成入庫
建立入庫紀錄。
---

### 回傳資料

透過 REST API 回傳：
* 設備端處理行為(如入/出庫)
* 相關聯單號(如進貨單/派工單...)
* 批號
* 序號
* 重量或數量
* 作業時間

## 出庫作業流程

## 出庫掃碼作業

### 掃描標籤
與今日任務清單進行比對驗證：

* 料品品項名稱
* 批號

### 完成出庫
建立出庫紀錄。
---

### 回傳資料

透過 REST API 回傳：
